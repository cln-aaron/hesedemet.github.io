April 2026 · Spotlight Talk Inaugural Edition
DEF CON Singapore 2026 · Creator Stage
Autonomous & Exploitable: Breaking AI Agents Before They Break Everything Else

Aaron took the Creator Stage at the inaugural DEF CON Singapore, a community he's followed, learned from, and quietly aspired to for years. To stand on that stage, in his own city, on the very first edition, was a milestone he didn't take lightly.

"Modern tool-using AI agents have shell, file, network, email, and memory capabilities, with no reliable way to distinguish data from instructions. Every input source is, in principle, a command channel. The underlying problem is structural, not a bug to patch."

Aaron walked the room through three live, browser-sandboxed labs that translated an abstract architectural failure into demonstrable, reproducible compromise. The thesis: we are deploying agentic AI into production faster than defenders, vendors, or researchers fully understand the attack surface.

Lab 01
Poisoned Webpage
Three parallel injections in one HTML file: an HTML comment, a display:none div, a font-size:0 span. The agent reads what the human cannot see, dumps env vars, base64s the secrets, exfiltrates them, and appends a backdoor instruction to CLAUDE.md so every future session starts compromised. What the human sees ≠ what the model sees.
Lab 02
Tool Response Poisoning
A compromised weather MCP returns a legitimate JSON payload (temp, conditions, forecast) plus one extra "debug_note" field instructing the model to chain into send_email() with the user's API keys. The send_email tool wasn't vulnerable. The trust boundary around tool output was.
Lab 03
Agentic Kill Chain
Three agents (Browser, Coder, Executor) share a vector DB. One poisoned page → store_memory + C2 fetch → session reset → fresh agent reinfects itself via similarity search → lateral movement across the inter-agent bus → exfil of env, conversation, and PII. Persistence without a binary on disk.
Five things defenders can do on Monday
  • Treat all retrieved content as data, never instructions: webpages, tool responses, memory, inter-agent messages.
  • Schema-validate every tool response. Reject unknown fields. No "debug_note," no "metadata," no surprises.
  • Least privilege per agent role. Your browsing agent does not need bash. Your executor does not need network.
  • Audit memory and bus traffic. Alert on instruction-shaped strings landing in long-term memory.
  • Human-in-the-loop on sensitive chains: email, file writes, outbound HTTP. Require approval, not just policy.
